# 비지도 학습 기반 독버섯 이상 검출
Kaggle Mushroom Classification 데이터셋을 활용하여 정상(식용) 데이터만 학습하고, 이상치(독버섯)를 탐지하는 비지도 학습 기반 예측 모델 프로젝트
<br/>

## 프로젝트 목적
- **이상 탐지**: 정상 데이터만으로 모델을 훈련시키는 비지도 학습 파이프라인을 구축
- **임계치 변경**: 모델이 출력하는 이상치 점수의 기준선을 조절하여 독버섯을 탐지하는 재현율 변화를 관찰
<br/>

## 데이터셋 구조
- **클래스**: 식용 버섯('e', 정상) / 독버섯('p', 이상치)
- **데이터 분할**:
  - *Train Set*: 식용 버섯 데이터의 80%만 사용하며 모델 학습
  - *Test Set*: 나머지 식용 버섯 20% + 독버섯 데이터 전체를 합쳐서 검증 데이터셋 구성
<br/>

## 전처리 및 모델링
- **전처리**: 문자열로 된 모든 범주형 변수를 모델에 입력할 수 있도록 `pd.get_dummies(X, drop_first_True)`를 통해 원-핫 인코딩 처리
- **모델**: 변수가 많은 고차원 데이터셋 특성을 고려하여 연산이 가볍고 차원에 강한 Isolation Forest 알고리즘 사용
<br/>

## Challenge
### [Scikit-Learn Isolation Forest]
- **음수 점수 매커니즘**: `score_samples()`는 단순한 확률(0~1)이 아니라, 확률 밀도 함수(PDF)의 로그값을 반환하기 때문에 결과가 대부분 음수로 출력
- **대소 관계 파악**: 알고리즘 특성상 값이 작을수록 고립도가 높은 이상치(독버섯)을 뜻하고, 값이 클수록(0에 가까운 음수일수록) 정상(식용버섯)을 뜻함
- **로직 검증**: 이에 따라 `anomaly_scores < custom_threshold` 조건문을 통해 이상치를 -1로, 정상치를 1로 매핑. 또한 성능 평가 함수인 `roc_auc_score` 역시 내부적으로 "값이 클수록 정상(1)"으로 판단하므로 별도의 부호를 뒤집지 않아도 올바르게 산출됨을 확인
<br/>

 ## 임계치별 성능 비교
 `score_samples`로 추출한 이상치 점수의 기준선을 50%, 70%, 80%으로 변경하며 오차 행렬 지표로 비교
 - **임계치 50%**: 전체 정확도는 높으나 독버섯 재현율이 0.58에 불과하며 전체 독버섯 중 1,663개를 미탐
 - **임계치 70%**: 식용 버섯 오진은 적으나, 독버섯을 809개 놓쳐 통과시킴
 - **임계치 80%**: 독버섯 미탐은 423개로 줄었으나, 멀쩡한 식용 버섯 313개를 독버섯으로 오진함
 - **최종 임계치 75%**: 독버섯 재현율 84%을 달성하여 미탐율 613개로 줄이고, 정상 식용 버섯 오진도 265개 수준으로 전체 정확도 82%, ROC-AUC 0.8288을 기록
